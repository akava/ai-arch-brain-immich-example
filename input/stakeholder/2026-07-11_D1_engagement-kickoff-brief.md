# D1 kickoff brief — Immich managed-hosting scale-up (simulated)

The client (a fictional managed-hosting operator) is evaluating Immich as the basis of a managed
photo-platform offering. Session D1 is a documentation-based architecture overview review.

## Questions for this session

1. What are Immich's runtime components, and what does each do? Which are stateful?
2. What are the integration points between components (protocols, queues, data stores)?
3. What is the technology stack, and where does the project document deliberate design decisions?
4. What operational characteristics are documented (jobs, scaling posture, hardware needs)?
5. What does the documentation say — or conspicuously not say — about running Immich beyond a
   single node?

## Engagement arc (for context)

- **D1** (this session): architecture overview review → baseline picture, first risks.
- **D2** (planned): machine-learning and media/storage pipeline deep dive.
- **D3** (planned): managed-hosting scale-up design — availability, ML throughput, storage
  strategy — culminating in a pilot enablement package.

## Ground rules

Evidence base is public first-party Immich documentation only. Gaps are surfaced as open
questions, never filled with assumptions.
