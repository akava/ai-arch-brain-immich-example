# Lab

Exploration and temporary work: prompt experiments, rough ideas, spikes, temporary notes, and early drafts not yet formalized.

This space is useful for thinking, but it is **not authoritative**.

## Key folders

- `researches/` — external/background research notes and surveys feeding decisions and specs (e.g. datastore engine-mode comparisons, feature-flag fundamentals, naming-convention surveys, arch-description frameworks).
- `architecture_playback/` — narrative walk-throughs of a scenario or read/write path used to pressure-test the architecture.
- `stakeholders/` — work-in-progress stakeholder and programme-people mapping.

## Rules

- Nothing in `lab/` is a deliverable or a source of truth.
- When an idea matures, promote it: structured outputs go to `core/artifacts/`, decisions to `core/decisions/`, approved delivery to `factory/`.
- Do not reference `lab/` files as approved architecture.
