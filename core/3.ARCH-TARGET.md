# 3.ARCH-TARGET.md — Target Architecture

## Purpose

`3.ARCH-TARGET.md` is the source of truth for the **target architecture** — the system rules this brain validates work against: reference architecture, sanctioned technologies, approved patterns, NFR baselines, and conventions. It defines what "good" architecture looks like here.

No structured output may contradict this file. Agents check here first (reuse before create).

The as-is counterpart is `core/2.ARCH-BASELINE.md`; phase movement lives in `core/transitions/`.

**Altitude:** scoped to system, data, and integration architecture — plus delivery and operating constraints that bind the system design (§5). Detail owned by other brains is referenced, not duplicated (see [Related brains](#related-brains)).
